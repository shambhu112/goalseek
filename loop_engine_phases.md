# Loop Engine Phases

Phases execute in order within each iteration: **READ_CONTEXT → PLAN → APPLY_CHANGE → COMMIT → VERIFY → DECIDE → LOG**, then loop back to READ_CONTEXT.

| Phase Name | Activities  | Files Created | Files Updated  | Coding Agent Called | Context Passed to Coding Agent |
|---|-----|---|---|--|---|
| **READ_CONTEXT** | Reads git history (recent 20 commits) and diff summary; enumerates all writable/visible files from manifest and hashes each; loads latest 5 results from history; loads active directions for this iteration | `runs/{iteration}/git_before.txt`, `runs/{iteration}/env.json` | `logs/state.json` | No | — |
| **PLAN** | Builds planning prompt from context, iteration info, and directions; calls AI hypothesis provider (`plan()`) to generate a plan with title and list of files it intends to modify; handles provider errors by short-circuiting to LOG | `runs/{iteration}/prompt.md`, `runs/{iteration}/plan.md`, `runs/{iteration}/provider_output.md` | `logs/state.json` | Yes — `provider.plan()` | Prompt containing: project name, readable/writable/generated/hidden scope patterns, last 5 iteration outcomes and metric values, active goal directions, stagnation flag (if 3+ consecutive non-kept outcomes); entire project root directory is added via `--add-dir` |
| **APPLY_CHANGE** | Ensures git working tree is clean; calls provider `implement()` to apply changes directly to disk; detects all changed files; validates no changes are out-of-scope (scope violations trigger an immediate revert commit); snapshots `experiment.py` if it exists | `runs/{iteration}/experiment.py` (snapshot) | Any files modified by the provider on disk; `logs/state.json` | Yes — `provider.implement()` | Prompt containing: iteration number, the full approved plan text from the PLAN phase, writable path patterns, generated path patterns, hidden path patterns, and editing constraints; entire project root directory is added via `--add-dir` with `--permission-mode acceptEdits` |
| **COMMIT** | Stages all changed files; commits them with message `"experiment: {plan_title}"`; records lines of code changed via `git show --numstat` | *(none — git commit only)* | `logs/state.json` | No | — |
| **VERIFY** | Runs all verification commands defined in the project manifest, each in its designated working directory; stops on first failure; extracts a scalar metric from output (JSON pointer, stdout regex, or stderr regex) | `runs/{iteration}/verifier.log`, `runs/{iteration}/metrics.json` | `logs/state.json` | No | — |
| **DECIDE** | Checks metric against pass/fail thresholds; compares metric to retained best using configured direction (maximize/minimize) and epsilon; accepts ("kept") or reverts the commit; on tie, prefers fewer lines changed; reverts via `git revert --no-edit` if rejected | *(none — git revert only if rolling back)* | `logs/state.json` | No | — |
| **LOG** | Captures final git state (`git show --stat`); generates human-readable results discussion; builds a `ResultRecord` with all iteration metadata; appends it to the append-only results log; regenerates the rolling history summary; resets iteration state and advances the counter | `runs/{iteration}/git_after.txt`, `runs/{iteration}/results_discussion.md`, `runs/{iteration}/result.json`, `runs/latest/history.json` | `logs/results.jsonl` (appended), `runs/{iteration}/provider_output.md` (final version), `logs/state.json` | No | — |
